## Recommended implementation path

If the target service accepts JSON:
1. Log in with `POST /api/v2/os/login`
2. Save the bearer token
3. Pull yearly overview from `GET /api/v4/os/clientoffice/diary/year`
4. Pull per-period detail from `GET /api/v4/os/clientoffice/diary/subjects`
5. Normalize subjects and scores into your destination schema

If the target service already consumes spreadsheets:
1. Log in with `POST /api/v2/os/login`
2. Call `GET /api/v4/os/clientoffice/diary/subjects/excel` or yearly export
3. Parse XLSX
4. Push rows into the destination service
